The next few examples will test the following code:
class Calculator
 attr_reader :fav_number, :silly_humans
 def initialize
  @fav_number = 8
  @silly_humans = 15
 end

 def add(num1, num2)
  num1 + num2
 end

 def PI
  Math::PI
 end

 def is_math_the_best_subject?
  true
 end

 def why_math
  "because it is cool"
 end

 def sillier
  @silly_humans += 5
 end
end

class MiniCalculator < Calculator

Run the RSpec test in terminal. $ rspec $FILENAME
Show two ways to write a pending test for a method that adds two numbers together. Method 1:
describe Calculator do
 context "#add" do
  it "adds two numbers together"
 end
end

Method 2:
describe Calculator do
 context "#add" do
  it "adds two numbers together" do
   pending
   my_calculator.add(1, 5).should eq 6
  end
 end
end
When you write a test should you say it "should add two numbers together" or it "adds two numbers together" It is better to say it "adds two numbers together" because should does not add anything and just takes up space.
Write an example (in RSpec, tests are called examples) to verify that the is_math_the_best_subject? method returns true. context "#is_math_the_best_subject?" do

The code examples thus far have been very redundant and a new instance of the Calculator class has been made in almost every example. What are two ways to dry out this code. 1. Use a let block to create an alias for Calculator.new
