Chapter 7: Treat Everything Like an Object

Question Answer
Use the code below to define a word_count method.
class Document
 def initialize
  @contents = "I love programming and that is the new story of my life"
 end

 def words
  @contents.split
 end
end 		def word_count
words.size
end

Two critical learning points from this example:
1. The words method is used as if it is a variable
2. self.words.size could have been used instead of words.size, but it was implicit that this method was being called on self, so an explicit reference was not necessary. When explicit references are not provided, Ruby assumes you meant to call the method on self.

If to_s is defined as a method, it will print out your custom message instead of the object reference and does not need to be explicitly called.

Chapter 9: Write Specs!

Question Answer
We will be testing the following example in this section:
class Document
 attr_reader :title, :author, :content
 def initialize(title, author, content)
  @title = title
  @author = author
  @content = content
 end

 def words
  content.split
 end

 def word_count
  words.length
 end
end 		Exciting!
Write a test that makes sure the Document class properly returns the contents. describe Document do
 it "should hold on to the contents" do
  text = "The cow jumped over the moon"
  book = Document.new("Cow", "Proust", text)
  book.content.should eq text
 end
end
Write a test that makes sure the document content includes some key words. This code should also go in the describe block:
it "the content should include the words" do
 text = "The cow jumped over the moon"
 book = Document.new("Cow", "Proust", text)
 book.content.include?("moon").should eq true
 book.content.include?("cow").should eq true
 book.content.include?("jumped").should eq true
end
Write a test that makes sure the word_count method is working properly. it "should return the right length of words for the contents" do
 text = "The cow jumped over the moon"
 book = Document.new("Cow", "Proust", text)
 book.word_count.should eq 6
end
Refactor the prior 3 examples to make the code less redundant. describe Document do
 before :each do
  @text = "The cow jumped over the moon"
  @book = Document.new("Cow", "Proust", @text)
 end

 it "should hold on to the contents" do
  @book.content.should eq @text
 end

 it "the content should include the words" do
  @book.content.include?("moon").should eq true
  @book.content.include?("cow").should eq true
  @book.content.include?("jumped").should eq true
 end

 it "should return the right length of words for the contents" do
  @book.word_count.should eq 6
 end
end
Write a test to make sure a number in Social Security Number format is included in the content of the following document:
text = "the man's SSN was 123-12-1234"
ssn_book = Document.new("Serial Killer", "FBI", text) 		it "should have numbers in social security number format" do
 text = "the man's SSN was 123-12-1234"
 ssn_book = Document.new("Serial Killer", "FBI", text)
 ssn_book.content.should match(/\d{3}-\d{2}-\d{4}/)
end
Make sure that the content of the cow jumps over the moon example that was used in earlier examples does not include the word San Francisco. it "should not include words that don't belong in the content" do
 text = "The cow jumped over the moon"
 book = Document.new("Cow", "Proust", text)
 book.content.should_not include("San Francisco")
end
What file should the RSpec test that test the Document class be saved in? document_spec.rb
The convention is to name the file the class name followed by _spec.rb
How do you isolate tests for a given class if that class relies on other classes? Stubs and mocks. A stub is an object that implements the same interface as a supporting class to the class that is being tested, but returns formulated answers when it is called.
